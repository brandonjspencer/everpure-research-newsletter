# Everpure deck pipelines

## 1) Deck normalization

Files generated by `everpure_deck_ingest.py`:
- `deck_details.json`: one normalized record per unique Google Slides deck
- `deck_week_map.json`: map from `file_id` to associated Everpure weeks
- `deck_summary.json`: high-level deck coverage stats

Run manually:

```bash
python everpure_deck_ingest.py \
  --data-dir ./output \
  --local-artifact-dir ./deck_artifacts
```

Supported local artifact names:
- `<file_id>.pdf`
- `<file_id>.pptx`
- `<file_id>.txt`
- `<file_id>.json`
- `<file_id>.html`

## 2) Deck PDF content ingestion

This extracts text from exported deck PDFs named by Google file ID.

Outputs:
- `deck_content.json`
- `deck_content_summary.json`

```bash
python everpure_deck_content_ingest.py \
  --data-dir ./output \
  --pdf-dir ./deck_artifacts
```

## 3) Google-authenticated fetch path

This pulls Slides metadata plus PDF/PPTX exports into `./deck_artifacts`.

OAuth access token path:

```bash
python everpure_google_fetch.py \
  --data-dir ./output \
  --artifact-dir ./deck_artifacts \
  --access-token "$GOOGLE_OAUTH_ACCESS_TOKEN"
```

Service account path:

```bash
python everpure_google_fetch.py \
  --data-dir ./output \
  --artifact-dir ./deck_artifacts \
  --service-account-json ./service-account.json
```

Optional domain-wide delegation impersonation:

```bash
python everpure_google_fetch.py \
  --data-dir ./output \
  --artifact-dir ./deck_artifacts \
  --service-account-json ./service-account.json \
  --subject user@yourcompany.com
```
